# CS210ProgrammingLanguages

1- What does the project do -

The project is a buying tracker for a store. This will track the frequency of what is bought and depending on the option selected will display the number or a Histogram form of the purchases. The last option allows you to find an item and tell you how many times it was purchased. After the creation of the class it will also saved a Frequency.dat file with all of the data. 

2- What was done well -

The project that I was most sucessful on was the basic coding of the class. Creating the frequency.dat file on creation also saved a few steps. This allowed for a simplistic creating the class and writing the file instead of setting another time to create the Frequency.Dat

3 - What could enhance your code -

One option that could have made the code better was a little more defined search option. Setting the search to not be case sensitive would be easier to search. Adding items to the file would also be beneficial to allow for a wider range of useage. 

4- Most challenging pieces -

I struggled with the while statements to get the histogram to display correctly at the start. Took a little time to get the nested statements in there to work properly. I also realized I made an error while loading the file. I had made an error in loading the file and the program instantly stopped. I put a try/catch in there to catch my error. 

5- Transferable skills -

OOP is the most transferable skill that can be done. This allows for easier updating and adding to code. If I needed to create another class like CostPerItem, some of the coding is already done. 

6- Maintainable, readable, and adaptable -

What made this Maintainable was creating the multiple classes with different files. The classes were made discriptive so that they could be looked at and changed where needed. The main file was kept as minimal as possible that the classes were defined enough so they could be edited and added too.
